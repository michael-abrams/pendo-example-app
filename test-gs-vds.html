APP-159523 — Goldman Sachs: host position: static !important vs guide layout

What this page proves. GS's site forces position: static !important. That can only affect a Pendo guide if the guide renders in the host light DOM (host CSS does not pierce a shadow root). This page renders the SAME building-block guide mock twice — once in light DOM, once inside an attached shadow root — so you can see exactly which path GS is on.

Open the console — the logger prints computed position + bounding box for every subject, and flags which ones are inside a shadow root.

1) Guide mock in LIGHT DOM

Host CSS reaches these nodes. Under ?gscss=*, expect the close X to drop into flow (top-left) and the polyfill button to lose centering. The native-flex button should stay centered.

2) Guide mock in SHADOW DOM

Host CSS cannot pierce the shadow boundary. If GS's real guide renders here, the host rule is a red herring and the bug is elsewhere. Everything should stay correct regardless of the toggle.

3) WORKAROUND probe — polyfill child forced with inline !important

If GS is on the light-DOM polyfill path, a Pendo-side fix that emits position:absolute !important inline on layout elements should beat even * { position: static !important } (inline !important outranks selector !important). This stage applies that fix in light DOM — expect it to stay centered.
